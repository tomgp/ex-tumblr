“

The standard mathematical definition of the Cartesian product is: A × B = {(a,b) | a ∈ A and b ∈ B }

Python’s list comprehensions let Norvig express this function in essentially the same notation as the standard mathematical definition:

def cross(A, B):
return [a b for a in A for b in B]

”—

Unit testing in Coders at Work

Seeing this piece of code was kind of a Damscene moment for me wrt Python. The kick I needed to start using it for stuff in ernest.

One week in and I already feel that I’m as productive as I have been with Perl which I’ve been using with reasonable regularity for a good 4-5 yrs. Some of the constructions have taken me a little effort to get used to (still getting caught by things like having to say len(list) rather than list.length and occasionally adding ; to the end of lines) but I can already see I’m going to miss the way modules, classes and objects work when I go back to using something else. Particularly the elegance (as noted above) of Python’s list comprehensions is going to be a real wrench to leave behind when I have to return to PHP next week.
