arguments for using noncontiguous in cartograms

Because I couldn’t fit an explanation in a tweet.

non contiguous cartogram

External image

contiguous cartogram

External image

Arguments for using noncontiguous cartograms from Judy Olson (1978) :

the empty areas, or gaps, between observation units are meaningful representations of discrepancies of values, these discrepancies generally being a major reason for constructing a cartogram

…

only the discrete units for which information is available and only the lines which can be accurately relocated on the original map appear on the noncontiguous cartogram

…

recognition of the units represented is relatively uncomplicated for the reader

In spite of their legibility problems I think fully contiguos cartograms (or squishy maps as I tend to think of them) are the most common type we see these days for a couple of reasons

  • They’re visually arresting i.e. “what the hell am I looking at?”
  • The algorithm for creating them is quite interesting. Classic tech led visualisation.

Also: contiguous cartograms will usually trade off some representational accuracy in order to keep eveything touching in the right place non-contigs needn’t do this.

Also: Non-contig readablity is better because of the spaces between the shapes, also because the position of features as well as their shape can be retained

So: Shape recognition and area estimation are both better in non-contig, these are probably the two main reasons for using a cartogram in the first place.
